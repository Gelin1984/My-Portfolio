Responsive Website

1.7: Bootstrap Components & Plug-ins

Learning Goals

  • Link Bootstrap file and describe use of Bootstrap framework, components, and plugins
  • Apply Bootstrap and customize Bootstrap glyphicon components for your Contact page
  • Apply and customize a Bootstrap collapse plugin to create an Accordion
  • Troubleshoot HTML, CSS, and Bootstrap templates

 Estimated Read Time: 30 Minutes.

Introduction

In the last Exercise, you got your website up to speed by creating a navigation bar from scratch and applying it across most of your webpages. Way to go! Now that you’ve got the basics of HTML and CSS down, we’re going to get fancy by adding components from a frontend framework called Bootstrap, which you'll learn all about in this Exercise.

In addition to learning how to use the Bootstrap framework, you'll learn how to customize it via "documentation." Ready to get started?

Cleaning Up Your CSS

In Exercise 1.4: CSS Layout & Box Model, we mentioned that a CSS best practice is to have one unified CSS file for your entire website rather than one per page. If you created a separate contact_styles.css file, now is the time to put its contents inside styles.css.

To keep your styles for different pages organized in a single file, you can use comments to group the code that belongs to specific pages together like this:

/**** Contact page ****/

Don't worry about the About page for now. We're going to take care of that one later.

If you created an element with the class .container on your contact page, you should either rename it (in both your CSS and HTML) or delete the stylings completely. Bootstrap uses a class with the exact same name and giving it your own styling might mess up your layout. We'll discuss this in more detail in Exercise 1.9: Responsive Web Design.

Inside the <head> of your Contact page, you'll also have to change the link to the CSS file to point to the styles.css file since we removed contact_styles.css completely.

What is Bootstrap?

Bootstrap is a frontend framework that contains HTML and CSS design templates for typography, forms, buttons, navigation, and other user interface components, as well as JavaScript extensions or plugins. Templates make our lives easier because they can add automatic styling and functionality.

A framework is a collection of tools for creating websites and web applications. Frameworks exist for both frontend and backend web development. For example, in a few weeks, you’ll learn about Rails (as part of Ruby on Rails), a backend framework for web applications. Bootstrap’s documentation is a user guide for all of Bootstrap’s capabilities and functions.

Bootstrap is a preferred web development framework for many reasons. First, it's open source and available on GitHub. Developers are encouraged to participate in the project and make their own contributions to the platform. Bootstrap also makes it easy to implement responsive design through its built-in grid system (Exercise 1.9: Responsive Web Design) and contains mobile-first styles, ready to use for modern browsers. Bootstrap also contains many free components, such as glyphicons.

To use Bootstrap components and plugins, we need to link Bootstrap CSS and JS files to our Contact page. To make this easier, we're going to link Bootstrap through a content delivery network (CDN). What this means is that, instead of using a Bootstrap file on your local computer, we're going to use Bootstrap components from a remote server.

Add the following code in the <head> tag of each HTML file (if it's not there already):

<!-- Latest compiled and minified CSS -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/twitter-bootstrap/3.3.7/css/bootstrap.min.css">

Remember that CSS is cascading. This means that the link to your styles.css file needs to be under the link to the bootstrap.min.css file. Otherwise, your stylings might be overwritten by Bootstrap.

Remember
You might remember us mentioning that Bootstrap has normalize.css already included. Because you're now adding Bootstrap to all files of your project, you should delete your previous link to normalize.css, as well as the file itself.

It's good for you to keep in mind that normalize.css exists and, if you should ever again create a website from scratch (meaning without using frameworks), you should probably relink normalize.css.

Bootstrap also has some really handy JavaScript components, so let's go ahead and add the JavaScript part of Bootstrap, as well.

At the bottom of your HTML, right before the </body> closing tag, add this script tag (you can add it under the comment "Bootstrap core JS" in your index.html file):

<script src="https://cdnjs.cloudflare.com/ajax/libs/twitter-bootstrap/3.3.7/js/bootstrap.min.js"></script>

What is JavaScript?
HTML is the markup language that displays all your content. CSS is used to style this content and create flashy layouts. Both languages are supported by all web browsers and can be interpreted by them to display websites. There is one more language, however, that's supported and interpreted by all web browsers by default nowadays: JavaScript. It used to be a language some browsers would deactivate by default, but over the years, it's become a web standard just like HTML and CSS.

JavaScript lets you make your website dynamic. That means it lets you animate features, react to user input, dynamically add or remove data, or do any other sort of logical processing. It lets you turn your website into a complex application, such as a to-do list, a text editor, or even a game.

We'll be teaching you more about JavaScript in a few Exercises.

For Bootstrap's JavaScript to work properly, you'll need to add a script tag with a link to jQuery before the link to the bootstrap.min.js. That's because Bootstrap is using some features from jQuery, and it can only access those features after jQuery has already been loaded. (Keep in mind an HTML document always loads from top to bottom.)

Here's the jQuery link:

<script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.2.1/jquery.min.js"></script>

What is jQuery?
jQuery is a huge library of pre-programmed JavaScript functions. It was developed to make our lives much easier when writing JavaScript. It has a lot of functionality for very common tasks people try to accomplish using JavaScript. For example, if you wanted to select an HTML element using its ID with JavaScript, the code would look like this: document.getElementById("thisIsAnId"). With jQuery, the same thing would look like this: $("#thisIsAnId").

All together, it could look like this:

  <!-- ============================= -->
  <!-- All your JavaScript comes now -->
  <!-- ============================= -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.2.1/jquery.min.js"></script>
  <!-- Bootstrap core JS -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/twitter-bootstrap/3.3.7/js/bootstrap.min.js"></script>
  <!-- Can place script tags with JavaScript files here -->

</body>

Don't forget to do this on each of your own pages, as well.

Why do I put the JavaScript at the bottom and not in the head?
There are several reasons for this. Putting your JavaScript files at the bottom of your HTML will ensure the rest of your website loads before the JavaScript files are downloaded and executed. This will not only speed up the load time of your website significantly, it's also necessary in many circumstances (when not using jQuery, for instance). Once your JavaScript file is loaded, it's executed by the browser. If your JavaScript function attempts to select or modify an HTML element that hasn't been loaded yet, it will raise a JavaScript error and possibly break the functionality of your site. (Note: in jQuery, best practice dictates that all functions should be run inside a document.readyblock to prevent this from happening, but this will be explained in a later Exercise.)

Bootstrap Components: Glyphicons

Let's add a glyphicon for each contact detail on your Contact page, including your email address, home address, and phone number. Glyphicons, a component of Bootstrap, are helpful as they can act like visual bullet points and break up text. Because they're applied to <span> elements, glyphicons can be inserted easily into the middle of a line or paragraph.

Note!
Span elements have a display value of inline, which means they'll be included as part of the existing line of text rather than being broken up into a new one (the way block elements such as <p> or <div> are).

Glyphicons are not actually images. They're an icon font—a custom font that uses glyphs instead of letters. (Remember Wingdings or Webdings? While that might bring back memories of the 90s, it’s still the same idea!)

Go to the Bootstrap documentation for components and pick at least three glyphicons you want to include to help draw your visitor to key information on your Contact page. 

Identify the name of each glyphicon. Then, use the following code to insert it. Put this <span> inside the element containing your text, such as a <p> or <li>.

<span class="glyphicon glyphicon-nameofyourglyphicon"></span>

The first “glyphicon” is a generic Bootstrap class that contains styling common to all glyphicons. The second class is the specific glyphicon. Replace “nameofyourglyphicon” with the glyphicon you'd like to use.

Check out this example of how icons were used in a professional Contact page. (Although these icons aren't glyphicons, you can use them to create a similar effect.)

Glyphicon Transitions and Effects

You can add transitions to glyphicons using pseudo-classes just as you did for the hover effect on your links. Refer to Exercise 1.6: Navigation to add hover properties to your glyphicons. The reason we can add these hover properties is thanks to glyphicons being interpreted as text rather than as images.

Bootstrap Plugins: Accordion

You may have some additional content about your work or services that doesn't fit on your About page. Create a new file for a Frequently Asked Questions page titled faq.html. Insert the required HTML header tags, then add a header and some <div> sections.

There are many ways to organize questions and answers on a FAQ page. You can see Etsy’s former FAQ page as an example below. Etsy used to have a unique accordion effect that allowed you to expand and collapse content.

We'll be implementing a similar effect in our FAQ page. We'll create this accordion using a plugin packaged in Bootstrap called collapse. Read the documentation and see a demo of collapse here.

Think of five questions visitors may ask about you when they come to your page that aren't answered on your About page. Some suggestions:
- Why did you decide to become a web developer?
- What is your design process?
- Who are your clients?
- What is your average project turnaround?

Write down your questions and your responses in a separate word processor document for now. You'll move your text into an accordion shortly.

Debugging

Another common reason why functionality on your page might refuse to work is due to JavaScript or CSS files being in the wrong order. Make sure you add the jQuery link before you load any other JavaScript files including the Bootstrap JavaScript file, as Bootstrap is dependent upon it. Do the same for your CSS files. Here, you need to be careful to always place the link to your framework first (such as bootstrap.min.css) and your custom stylesheets following it. Otherwise, Bootstrap will simply overwrite your custom styles.

Whenever something goes wrong, first check whether you have the right libraries linked (most of which you can find through the documentation).

Styling Your Accordion

Style your accordion with CSS so it doesn't look like the default Bootstrap version. What kind of font and text will your answers be? Modify the panel-body class in your CSS file. Can you give your headings a different background color? Modify the background-color property of the panel-heading class. What about your heading text? Modify the panel-title styles in your CSS file.

You can also add glyphicons to make the headers more prominent, as well. We added a "+" glyphicon using this code below, but you can return to the Bootstrap documentation to choose your own.

<a data-toggle="collapse" data-parent="#accordion" href="#collapseOne"><span class="glyphicon glyphicon-plus"></span>Collapsible Group Item</a>

Summary

This Exercise has given you the opportunity to focus more on Bootstrap, which has given you a plethora of options to truly make your website your own. While we've gone over several of those options within the text, we encourage you to dig deeper into the world of Bootstrap to see all of what you can do.

Learn More

Throughout this Exercise and the rest of the frontend development unit, you may want to take a look at the Bootstrap documentation in more detail to understand its full capabilities. Some of what you'll find can get pretty technical and seem too overwhelming for your current skill level, but don't worry! We don’t expect you to read and understand everything overnight. Reading documentation, just like debugging, is a skill that you'll develop over time!
